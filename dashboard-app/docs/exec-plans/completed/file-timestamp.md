# サイドバーにファイル最終更新日時を表示

- **状態**: completed
- **作成**: 2026-04-11
- **完了**: 2026-04-12
- **生成者**: Planner Agent
- **検証**: Evaluator AC-1〜11 全 pass / code-reviewer verdict=minor-findings（Finding 1, 4 対応済み）

## 目的

サイドバーのファイルツリーで、各ファイル名の下にそのファイルの最終更新日時を小さく表示し、ユーザーがどのファイルが最近更新されたかを一目で把握できるようにする。

## 仕様（WHAT）

### BE の変更

- `/api/files` が返す `FileNode`（`type: 'file'`）に、ファイルの最終更新日時を表すフィールドを追加する。
- ディレクトリノード（`type: 'directory'`）には日時フィールドは不要。
- 日時の形式は ISO 8601 文字列（例: `"2026-04-10T14:30:00.000Z"`）。
- 既存のフィールド（`type`, `name`, `path`）はそのまま維持する。

### FE の変更

- サイドバーのファイルツリーで、各ファイル名の直下に最終更新日時をローカルタイムゾーンで人間が読みやすい短い形式で表示する（例: `2026/04/10 14:30`）。
- 日時テキストはファイル名よりも小さいフォントサイズで、控えめな色で表示し、ファイル名の視認性を妨げない。
- ディレクトリ名の下には日時を表示しない。

- [ ] AC-7: ファイルをクリックして本文を切り替えた後も、サイドバーの日時表示が引き続き正しく表示されている（`take_snapshot` で確認）

## 非スコープ

- 日時の相対表記（「3 時間前」「昨日」など）
- 日時によるソート機能
- 日時のツールチップ表示
- 日時フォーマットのユーザー設定
- ディレクトリに対する「最新の子ファイルの日時」集約表示
