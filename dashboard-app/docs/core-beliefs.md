# Core Beliefs / 黄金原則

> このファイルは、本プロジェクトで「設計判断に迷ったときの軸」と「同じミスを二度繰り返さないためのルール」を蓄積する場所です。
> エージェントが同じ失敗を2回したら、ここに1行追記してから直してください。
> ルールが安定したら、可能な限り**カスタムリンタ・テスト・型**に昇格させてください（プロンプトより仕組みが強い）。

---

## 設計判断の軸（不変条件）

### Legibility ファースト
- エージェントが**定量的に把握可能な**形式を優先する。
- 見えない暗黙知は存在しないものと扱う。判断はコード or docs に書き出す。

### 境界は厳格、内部は自由
- レイヤ間の依存方向、API スキーマ、データ形は厳密に守る。
- 各レイヤの**内部実装**は自由（ライブラリ選択、命名スタイルなど細部にはこだわらない）。

### 決定的な層を厚く、非決定的な層を薄く
- リンタ・型・テスト・スキーマ検証で守れるものはそちらで守る。
- プロンプトや AGENTS.md の文章は最小限にする。

### 1セッション1機能、1PR1目的
- 1度に複数の関心事を混ぜない。
